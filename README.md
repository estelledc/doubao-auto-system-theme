# 豆包网页版自动深色模式

让豆包网页版聊天页跟随系统深浅色，并补齐深色下常见的可读性问题。

> 非豆包官方脚本。当前版本覆盖主要聊天场景；豆包前端改版后可能需要继续适配。

## 为什么做

豆包网页版目前没有稳定可用的完整深色模式。晚上使用、阅读长回答或看代码时，常见问题包括：

- 左侧历史记录文字太暗，看不清会话标题。
- 回答正文还是黑字，落在深色背景上几乎不可读。
- 代码块出现白色顶栏、黑底黑字或语法高亮错色。
- 输入框、弹窗、菜单局部残留白底。

这个脚本的目标不是把页面简单反色，而是做一个“主题适配器”：先让豆包页面跟随系统深浅色，再精确修补漏出的浅色层和低对比文字。

## 做了什么

- 跟随系统：读取 `prefers-color-scheme`，系统切换浅色/深色时无需刷新页面。
- 驱动原生主题：同步 `html` 和 `body` 上的 `data-theme`、`theme-mode` 等主题属性。
- 侧栏可读：单独处理历史会话、搜索框、入口和用户信息区。
- 正文可读：修复 Markdown 标题、段落、列表、引用、表格和行内代码。
- 代码块可读：修复代码块外壳、顶栏、按钮、滚动条和 Prism 语法 token。
- 输入区和浮层：修复输入框、下拉菜单、弹窗、toast 等常见白底残留。
- 内置审计：在控制台运行 `__doubaoThemeAudit()` 可查看侧栏、正文、代码块对比度检查结果。

## 不做什么

- 不使用 `filter: invert()` 粗暴反色。
- 不全局改 `p/span/button`，避免破坏头像、图片、品牌色和页面控件。
- 不读取、保存或上传聊天内容。
- 不支持豆包聊天页以外的页面。
- 不承诺永久适配豆包未来所有 UI 改版。

## 安装

1. 安装浏览器扩展 [Tampermonkey](https://www.tampermonkey.net/)。
2. 从 Greasy Fork 安装脚本：搜索 `豆包网页版自动深色模式`。
3. 打开或刷新 [豆包网页版聊天页](https://www.doubao.com/chat/)。
4. 切换系统外观，页面会自动跟随浅色或深色。

也可以手动导入：

1. 打开 Tampermonkey 管理面板。
2. 新建脚本。
3. 粘贴 `doubao-auto-system-theme.user.js` 全文并保存。

## 适用范围

- 浏览器：Chrome + Tampermonkey。
- 页面：`https://www.doubao.com/chat/*`。
- 模式：只跟随系统深浅色，不提供手动强制开关。

## 反馈问题

如果遇到白块、黑字、看不清或页面改版失效，请提交 issue，并尽量附上：

- 浏览器版本和 Tampermonkey 版本。
- 系统当前是浅色还是深色。
- 豆包页面 URL。
- 问题区域截图。
- 控制台运行 `__doubaoThemeAudit()` 的输出。
